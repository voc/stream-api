## Origin stage
The origin machines run a Webserver that serves the playlists, segments and thumbnails generated by the transcoding stage to the edge servers.

### Upload Server
To receive the uploads from the upload-proxy in the transcoding stage, the origin stage runs the [upload-server](../cmd/upload-server/) binary which receives the uploads and stores them on disk.

It also processes the incoming stream playlists to provide a consistent HLS format with linearly increasing segments and appropriate discontinuity tags.

### Webserver
The origins run nginx to serve the local files for edge servers. It also serves as TLS reverse proxy for the upload-server.
